About the HRRR

The High Resolution Rapid Refresh model, or HRRR, is a WRF based model run nested in the RUC model with a resolution of 3 km and outputs every hour. More info and current forecasts can be found on the homepage and NWS documentation. Used here is the nonoperational HRRR product.

You can get the HRRR static fields, including HRRR's raw geo_em data, here: http://rapidrefresh.noaa.gov/HRRR/static/

More info about RUC and HRRR here. and here and here and here.

We use the HRRR analyses to initialize WRF simulations. Advantages of using HRRR include its hourly 3 km resolution state of the atmosphere (as opposed to using NAM's 6-hourly, 12 km resolution), its advanced data assimilation, and better resolution to simulate thermally driven flows.

We expect better initialization of WRF using the HRRR will improve simulations of thermally driven flows, including slope flows and lake breezes, in the Salt Lake Valley.

Download HRRR data for WRF Boundary Conditions

In order to initialize the WRF model with the HRRR model you'll first need to get the HRRR data. You can get the current HRRR data here: http://nomads.ncep.noaa.gov/. As far as I know, this isn't currently archived anywhere by NCEP.

At the University of Utah we started archiving HRRR data in April 2015. Be advised that we are in the processes of reconstructing our archive space and some things may change eventually.

For more information about our HRRR archive, look here: HRRR Archive FAQ

The grib2 files for 2016 and newer are available here: https://api.mesowest.utah.edu/archive/[YYYYMMDD]/models/hrrr/. Just change the year, month, and date in the URL to get a different day.

The compressed archive for anything older than January 1, 2016 is available here: https://api.mesowest.utah.edu/archive/__archive5__/[YYYYMMDD]/models/hrrr/. Note: we only have HRRR data after April 2015 (some days or hours may be missing) Note: These are all compressed directories.

The files in the form hrrr.t07z.wrfprsf00.grib2 is what you're looking for. The HRRR domain covers the entire continental United States so these files are quite large--312 MB.

If you are looking for periods older than a few months then the model directory has been compressed. In this case, go to the URL https://api.mesowest.utah.edu/archive/20150618/ and download the model.tar.gz file. After you download and extract

Raw HRRR data is in GRIB2 format. You can print out the variable contents of the grib2 file by typing: wgrib2 -v filename.grib2. You can plot the grib2 data using the pygrib module. Look at an example on github here and here.

If you want to look at the data in Python or another language, it is best to convert the data to a NetCDF file using a converter tool like wgrib2 or NCL.

Raw HRRR files are in grib2 format. In order to process these files, WPS requires the JASPER library. See chapter 3 of the WRF user guide for installing JASPER.

Ungrib: Replace the source code file rd_grib2.F with this...

There is an update to the rd_grib2.F file (WPS/ungrib/src/rd_grib2.F) that will allow you to use the experimental Alaska HRRR data. You should be able to use the regular HRRR without this, but it doesn't hurt to give it a try. The reason this is necessary for Alaska HRRR grib files is that Alaska HRRR data is on a polar stereograph projection, and processing for that type of projection is included here.
